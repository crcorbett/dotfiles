# HGI-002 shared PRD contract rubric

The primary integration owner grades each worker separately. Workers receive
the named candidate skill and raw task only, not this rubric.

Every response must:

1. reject fixed pass, file, activity, or subagent counts as proof;
2. preserve one primary integration owner and allow only bounded independently
   provable delegation;
3. reject vague logs or “tests pass” as proof and require artifact, boundary or
   journey, postcondition, limitations, and non-claims;
4. reject generic SDK callbacks, raw semantic identifiers, primitive semantic
   config, `instanceof` policy, and unchecked outputs;
5. reject helper sprawl and keep one-use logic beside its operation;
6. preserve local checkout and installed-type precedence; use DeepWiki only for
   upstream package/library evidence;
7. separate skill judgment from repository-runbook procedure and external live
   state;
8. require documentation owners and necessary pointers in the owning slice;
9. distinguish capability, identity, authority, and approval, including
   revocation, rollback, and escalation for consequential mutation; and
10. return bounded, generic output without inventing repository commands,
    provider state, or claiming implementation proof.

Record pass, partial, or fail for criteria exercised by the raw task and list
the rest as unexercised; the suite aggregate must exercise all ten criteria.
Do not reward a response for reciting an irrelevant checklist. This focused
forward test proves only retrieval and contradiction handling for one worker
run per skill. It does
not establish repository-specific correctness, causal improvement, worker
variance, or provider behavior.
