# Operational coverage matrix

Use this as a coverage check, not a style checklist. A row passes only when the visual, specification, and reviewer evidence satisfy its stated test.

| Embedded principle | Skill surface | Evidence required |
| --- | --- | --- |
| Image, word, and number form one argument | Question, annotation, direct labels | Reader can state question, comparison, evidence, and qualification from graphic alone. |
| Preserve dimensions and disclose loss | Data and transformation contract | Specification names grain, dimensions retained, aggregation/projection, and material loss. |
| Connect macro, micro, and context | Comparison and layout review | Name an overview, inspectable detail, and source/uncertainty context visible without a view switch. |
| Layer meaning by hierarchy | Hierarchy declaration and render review | Data dominates; guides, containers, and explanatory layers have quieter weight/value. |
| Repeated design enables comparison | Facet and eyespan checks | Relevant panels retain scale, order, mark, and interval; exceptions are disclosed. |
| Colour is a semantic budget | Colour necessity record | Every hue records a reading lost without it; neutral context and restricted accent are visible. |
| Narrative retains time/space context | Annotation and responsive plan | Event order and spatial relation remain recoverable, or sacrificed dimension and alternative are named. |
| Visual effect matches data effect | Integrity checks | Verify units, denominator, domains, interval, baseline, and visual/data effect where measurable. |
| Graphics require three competences | Competence gate | Identify domain meaning, statistical validity/uncertainty, and visual choice. |
| Marks and revision serve evidence | Mark-economy review | Explain each non-data layer as orientation, comparison, explanation, access, or provenance. |
| Form and target size are reasoning | Form-choice and target-size record | Explain chart/table/text choice; inspect intended, reduced, grayscale, and narrow sizes. |
| Candidate forms are generated before selection | Candidate-form record | Name data task, at least two forms, and why the selected form wins. |
| Editorial production is data work | Brief and rendered review | Name audience/use; reconcile visible values; verify primary evidence survives reduced attention. |
| Tables support exact lookup | Form-choice review | Use table/table-graphic when exact values and multiple lookups dominate. |
| Dual scales can create false correlation | Dual-scale gate | Align zero when relevant; document necessity; otherwise split panels or index variables. |
| Rules need visual judgement | Exception record | Every departure names competing evidence and reader benefit. |

## Independent review protocol

The builder may score the rubric but may not certify it. A separate reviewer inspects the rendered output and source data/specification, tries removing each accent and guide, and challenges every top score with visible evidence. A self-score is draft only.
